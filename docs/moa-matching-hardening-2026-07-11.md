# 1. Process name normalization specification

## Required semantics

A rule process name matches only when its normalized basename is exactly equal to the event process basename.

Do not:

• Use substring matching.
• Infer executable extensions.
• treat `cmd` as equivalent to `cmd.exe`.
• Resolve executable paths through the filesystem.
• apply host OS path semantics to telemetry from another platform.

Examples:

| Rule | Event | Platform | Result |
|---|---|---:|---:|
| `powershell.exe` | `powershell.exe` | Windows | Match |
| `powershell.exe` | `POWERSHELL.EXE` | Windows | Match |
| `powershell.exe` | `notpowershell.exe` | Windows | No match |
| `powershell.exe` | `mypowershell.exe` | Windows | No match |
| `cmd` | `cmd.exe` | Windows | No match |
| `cmd.exe` | `C:\Windows\System32\cmd.exe` | Windows | Match |
| `bash` | `/usr/bin/bash` | Linux | Match |
| `bash` | `/usr/bin/Bash` | Linux | No match |

## Exact algorithm

1. Require a string.
2. Strip leading and trailing whitespace.
3. Treat both `/` and `\` as separators because Windows telemetry may be analyzed on Linux.
4. Take the final nonempty path component.
5. Reject an empty result.
6. For Windows, normalize with `str.casefold()`.
7. For Linux, preserve case exactly.
8. Compare with `==`.

Use the rule platform for both operands. Matching has already established that `event.platform == rule.platform`, but using one authoritative platform avoids inconsistent normalization if malformed events reach the helper.

## Pseudocode

```python
SUPPORTED_PLATFORMS = {"windows", "linux"}


def normalize_process_name(name: str, platform: str) -> str:
    if platform not in SUPPORTED_PLATFORMS:
        raise ValueError(f"Unsupported platform: {platform}")

    if not isinstance(name, str):
        return ""

    value = name.strip()
    if not value:
        return ""

    basename = value.replace("\\", "/").rsplit("/", 1)[-1]

    if not basename:
        return ""

    if platform == "windows":
        return basename.casefold()

    return basename
```

`casefold()` is preferable to `lower()` for explicitly case insensitive comparison. For ordinary Windows executable names they produce the same result.

## Rule validation

At rule load time:

• Reject empty `process_name`.
• Reject values ending in `/` or `\`.
• Prefer requiring a basename in rule files, not a path.
• Do not silently add `.exe`.

Pre-normalizing the rule value is optional. It saves a small amount of repeated work but should not mutate the original detection dictionary because `Rule.to_dict()` exposes it through the API. Cache normalized values separately.

```python
class Rule:
```

# 2. Parent process matching specification

Parent process matching must use exactly the same normalizer and exact comparison.

No substring behavior is permitted. Therefore:

• `explorer.exe` does not match `internetexplorer.exe`.
• `explorer` does not match `explorer.exe`.
• `C:\Windows\explorer.exe` matches `explorer.exe` on Windows.
• `/usr/lib/systemd/systemd` matches `systemd` on Linux.
• Missing parent telemetry never matches a configured parent criterion.

# 3. Log source validation specification

## Security boundary

`validate_log_source()` must be the sole authority for source path validation.

The current architecture validates in the API, then invokes collectors whose inherited `validate_source()` performs weaker validation. That creates two policies and permits insecure direct collector invocation.

Recommended architecture:

1. Store source policy on each collector.
2. Make `BaseCollector.validate_source()` delegate to `core.source_validator`.
3. Every collector calls that method before opening anything.
4. The API may validate early for a clean HTTP 400 response, but collector validation remains mandatory as defense in depth.
5. The collector must use the returned resolved `Path`, never the original request path.

This keeps CLI, tests, future endpoints, and direct library calls protected.

## Fail closed allowed roots

An empty allowed-root list must not mean unrestricted filesystem access in production. That defeats the containment requirement.

Recommended policy:

• Default allowed roots: `["./logs", "./tests/fixtures"]` is unsuitable for production because tests should not be runtime policy.
• Production default: `["./logs"]`, resolved relative to the configuration file or application working directory.
• If `allowed_roots` is absent or empty, reject scans at startup or scan time.
• Tests may explicitly inject temporary roots.
• If unrestricted operation is truly necessary, require a separate explicit `allow_unrestricted_sources: true` setting. Do not overload an empty list with that meaning.

This is intentionally stricter than the partially implemented backward-compatible behavior.

## Extension allowlists

Use exactly the requested extensions:

```python
ALLOWED_EXTENSIONS = {
    "windows": frozenset({".log", ".xml", ".evtx"}),
    "linux": frozenset({".log", ".json"}),
}
```

Current extras should be removed:

• Windows `.json` is not in the requirement.
• Linux `.txt` is not in the requirement.

Suffix comparison is case insensitive because extension spelling is not a security identity:

```python
suffix = path.suffix.casefold()
```

## Maximum sizes

Recommended defaults:

• Per-file maximum: 100 MiB, `104857600` bytes.
• Maximum aggregate bytes for a directory: 500 MiB.
• Maximum files per directory scan: 1,000.
• Maximum recursion depth: 10.

The per-file limit alone is insufficient. A directory containing 100,000 files of 99 MiB each still creates a denial of service condition.

## Containment algorithm

1. Reject non-string and empty source values.
2. Reject unsupported platforms.
3. Require at least one configured root unless unrestricted mode is explicitly enabled.
4. Resolve every configured root with `Path.resolve(strict=True)`.
5. Require each root to be an existing directory.
6. Resolve the source with `Path.resolve(strict=True)`.
7. Use `Path.is_relative_to(root)`, not string prefix comparison.
8. Require containment within at least one resolved root.
9. Use `os.lstat()` on the submitted path to identify symlinks before following them.
10. Symlinks may be accepted only if their fully resolved target remains in an allowed root.
11. Reject anything that is not a regular file or directory.
12. Validate every directory entry independently before opening it.
13. Skip directory symlinks and do not follow links during traversal.
14. Validate extension, size, count, aggregate size, and relative recursion depth.
15. Return immutable validated paths or a scan manifest, not merely `True`.

`Path.is_relative_to()` avoids prefix bugs such as treating `/var/log-evil` as contained in `/var/log`.

## Special file handling

Use `stat` mode checks:

```python
mode = path.stat(follow_symlinks=True).st_mode

if stat.S_ISREG(mode):
    ...
elif stat.S_ISDIR(mode):
    ...
else:
    raise SourceValidationError(
        "Log source must be a regular file or directory"
    )
```

This rejects:

• Character devices.
• Block devices.
• FIFOs.
• Unix sockets.
• Other special filesystem objects.

Do not test readability by opening in text mode. Validation should use metadata. The parser chooses binary or text mode later:

• `.evtx`: binary parser.
• `.xml`, `.log`, `.json`: text parser with explicit encoding and error policy.

## TOCTOU limitation

Resolving and validating a path before opening it leaves a small race where an attacker with write access to the source directory could replace the file.

Implementable one-session mitigation:

• Reject symlink directory entries.
• Validate immediately before parsing.
• Open each file once.
• Compare `os.fstat(file.fileno())` against the validated `stat` result where practical.

Full race resistance would require descriptor-relative operations such as `openat` with no-follow flags. That is a separate hardening phase.

## Root and containment helpers

```python
def resolve_allowed_roots(
    configured_roots: list[str],
    allow_unrestricted: bool,
) -> tuple[Path, ...]:
    if not configured_roots and not allow_unrestricted:
        raise SourceValidationError(
            "No allowed log source roots are configured"
        )

    roots = []

    for value in configured_roots:
        root = Path(value).expanduser().resolve(strict=True)

        if not root.is_dir():
            raise SourceValidationError(
                f"Allowed source root is not a directory: {value}"
            )

        roots.append(root)

    return tuple(roots)


def is_contained(path: Path, roots: tuple[Path, ...]) -> bool:
    return any(path == root or path.is_relative_to(root) for root in roots)
```

## Single-path validation pseudocode

```python
def validate_log_source(
    source: str,
    platform: str,
    policy: SourcePolicy,
) -> Path:
    if platform not in ALLOWED_EXTENSIONS:
        raise SourceValidationError(
            f"Unsupported platform: {platform}"
        )

    if not isinstance(source, str) or not source.strip():
        raise SourceValidationError("Log source path is empty")

    submitted = Path(source.strip()).expanduser()

    try:
        submitted_lstat = submitted.lstat()
        resolved = submitted.resolve(strict=True)
    except FileNotFoundError as exc:
        raise SourceValidationError(
            "Log source does not exist"
        ) from exc
    except OSError as exc:
        raise SourceValidationError(
            "Log source cannot be resolved"
        ) from exc

    if policy.allowed_roots and not is_contained(
        resolved,
        policy.allowed_roots,
    ):
        raise SourceValidationError(
            "Log source is outside configured allowed roots"
        )

    if not policy.allowed_roots and not policy.allow_unrestricted_sources:
        raise SourceValidationError(
            "No allowed log source roots are configured"
        )

    mode = resolved.stat().st_mode

    if stat.S_ISREG(mode):
        validate_regular_file(resolved, platform, policy)
    elif stat.S_ISDIR(mode):
        validate_directory_source(resolved, platform, policy)
    else:
        raise SourceValidationError(
            "Log source must be a regular file or directory"
        )

    return resolved
```

Do not return configured root paths or raw user paths in API errors. That leaks filesystem structure. Put detailed paths in protected server logs if needed.

## File validation pseudocode

```python
def validate_regular_file(
    path: Path,
    platform: str,
    policy: SourcePolicy,
) -> int:
    mode = path.stat(follow_symlinks=True).st_mode

    if not stat.S_ISREG(mode):
        raise SourceValidationError(
            "Source entry is not a regular file"
        )

    if path.suffix.casefold() not in policy.extensions:
        raise SourceValidationError(
            f"Unsupported log extension for {platform}"
        )

    size = path.stat(follow_symlinks=True).st_size

    if size > policy.max_file_size_bytes:
        raise SourceValidationError(
            "Log file exceeds configured size limit"
        )

    return size
```

## Directory manifest pseudocode

A validated manifest is safer than validating a directory and letting the collector rediscover arbitrary files afterward.

```python
def build_scan_manifest(
    root: Path,
    platform: str,
    policy: SourcePolicy,
) -> tuple[Path, ...]:
    files = []
    total_size = 0

    for current, directories, names in os.walk(
        root,
        followlinks=False,
    ):
        current_path = Path(current)
        relative = current_path.relative_to(root)
        depth = len(relative.parts)

        if depth >= policy.max_recursion_depth:
            directories.clear()

        directories[:] = [
            name
            for name in directories
            if not (current_path / name).is_symlink()
        ]

        for name in names:
            candidate = current_path / name

            if candidate.is_symlink():
                resolved = candidate.resolve(strict=True)

                if not is_contained(
                    resolved,
                    policy.allowed_roots,
                ):
                    raise SourceValidationError(
                        "Symlink escapes allowed roots"
                    )
            else:
                resolved = candidate.resolve(strict=True)

            if not is_contained(resolved, policy.allowed_roots):
                raise SourceValidationError(
                    "Directory entry escapes allowed roots"
                )

            if resolved.suffix.casefold() not in policy.extensions:
                continue

            size = validate_regular_file(
                resolved,
                platform,
                policy,
            )

            files.append(resolved)
            total_size += size

            if len(files) > policy.max_files:
                raise SourceValidationError(
                    "Directory contains too many log files"
                )

            if total_size > policy.max_total_size_bytes:
                raise SourceValidationError(
                    "Directory exceeds aggregate size limit"
                )

    return tuple(files)
```

For the simplest one-session implementation, reject all file symlinks inside directory scans. Accepting safe in-root symlinks adds little value and complicates duplicate detection.

# 4. Regex precompilation specification

## Current state

`Rule._compile_regexes()` already:

• Compiles `command_regex`.
• Compiles `user_pattern`.
• Uses `re.IGNORECASE`.
• Caches compiled objects.
• Rejects syntax errors during `Rule` construction.

The engine uses the cache at `core/engine.py:255` and `core/engine.py:281`, but it retains fallback calls to `re.search()`. Those fallbacks should be removed. A configured regex without a cached compiled object is an invariant violation, not a reason to compile during event processing.

## Load-time behavior

For each regex field:

1. Require a string.
2. Reject empty patterns.
3. Reject patterns longer than 500 characters.
4. Parse and validate structural complexity.
5. Reject known dangerous nested repetition, including `(a+)+`.
6. Reject repetition nesting deeper than 3.
7. Compile once with established flags.
8. Cache under the detection field name.
9. If any regex fails, reject that rule.
10. In strict startup mode, fail application initialization if any rule is rejected.

Recommended exact values:

```python
MAX_REGEX_PATTERN_LENGTH = 500
MAX_REGEX_REPEAT_NESTING = 3
REGEX_FLAGS = re.IGNORECASE
```

Five hundred characters is ample for the existing rules, whose patterns are much shorter. A 1,000-character ceiling is defensible, but unnecessary for this project.

## Important correction about nested quantifiers

Only enforcing “depth no greater than 3” is not enough. `(a+)+$` has shallow nesting and is a classic catastrophic-backtracking expression.

Use both controls:

• Hard maximum repetition nesting of 3.
• Reject a repeated group if it contains another variable repetition and the outer repetition is variable.

Variable repetition means `*`, `+`, `{m,}`, or `{m,n}` where maximum differs from minimum. Fixed repetitions such as `{4}` are much safer.

Static analysis remains heuristic. Python `re` provides no execution timeout. These checks significantly reduce risk but do not mathematically prove linear runtime.

```python
def validate_regex_pattern(
    pattern: str,
    field_name: str,
    rule_id: str,

    if not isinstance(pattern, str):
        raise ValueError(
            f"Rule {rule_id}: {field_name} must be a string"

    if not pattern:
        raise ValueError(
```

A regex-based checker will not correctly understand every nested group. The stronger implementation should tokenize grouping and escapes rather than count raw parentheses:

```python
def estimate_repeat_nesting(pattern: str) -> int:
    group_stack = []
    group_contains_repeat = []
    maximum = 0
    escaped = False
    in_character_class = False
    index = 0

    while index < len(pattern):
        char = pattern[index]

        if escaped:
            escaped = False
            index += 1
            continue

        if char == "\\":
            escaped = True
            index += 1
            continue

        if char == "[":
            in_character_class = True
            index += 1
            continue

        if char == "]" and in_character_class:
            in_character_class = False
            index += 1
            continue

        if in_character_class:
            index += 1
            continue

        if char == "(":
            group_stack.append(False)
            index += 1
            continue

        if char == ")":
            contains_repeat = (
                group_stack.pop() if group_stack else False
            )
            quantifier = read_quantifier(pattern, index + 1)

            if quantifier is not None:
                parent_depth = sum(group_stack)
                current_depth = parent_depth + 1
                maximum = max(maximum, current_depth)

                if contains_repeat and quantifier.is_variable:
                    raise UnsafeRegexError(
                        "Nested variable quantifier"
                    )

                if group_stack:
                    group_stack[-1] = True

            index += 1
            continue

        quantifier = read_quantifier(pattern, index)

        if quantifier is not None:
            if group_stack:
                group_stack[-1] = True

            maximum = max(
                maximum,
                1 + sum(group_stack),
            )
            index = quantifier.end
            continue

        index += 1

    return maximum
```

This is still a conservative policy checker, not a complete regular expression parser. Document that explicitly.

```python
class Rule:
```

## Configuration validation

```python
def validate_log_source_config(config):
    settings = config["log_sources"]

    roots = settings["allowed_roots"]
    unrestricted = settings["allow_unrestricted_sources"]

    if not isinstance(roots, list):
        raise ConfigurationError(
            "log_sources.allowed_roots must be a list"
        )

    if not all(isinstance(root, str) and root.strip() for root in roots):
        raise ConfigurationError(
            "Every allowed source root must be a nonempty string"
        )

    if not roots and not unrestricted:
        raise ConfigurationError(
            "At least one allowed source root is required"
        )

    numeric_bounds = {
        "max_file_size_bytes": (1, 1024 * 1024 * 1024),
        "max_total_size_bytes": (1, 10 * 1024 * 1024 * 1024),
        "max_files": (1, 100_000),
        "max_recursion_depth": (0, 32),
    }

    for key, bounds in numeric_bounds.items():
        value = settings[key]
        minimum, maximum = bounds

        if (
            not isinstance(value, int)
            or isinstance(value, bool)
            or not minimum <= value <= maximum
        ):
            raise ConfigurationError(
                f"log_sources.{key} must be between "
                f"{minimum} and {maximum}"
            )

    if (
        settings["max_total_size_bytes"]
        < settings["max_file_size_bytes"]
    ):
        raise ConfigurationError(
            "Aggregate size limit cannot be smaller "
            "than the per-file size limit"
        )
```

Do not require allowed roots to exist during generic config parsing if deployment creates them later. Resolve and require them during application startup. Either policy is valid, but mixing the two creates confusing failures.

# Acceptance criteria

The implementation is complete when:

• Process and parent matching use normalized exact basename equality only.
• Windows matching is case insensitive.
• Linux matching is case sensitive.
• No regex is compiled during event processing.
• Invalid or unsafe regexes prevent strict startup.
• Every collector invocation applies the same centralized source policy.
• A resolved source must remain inside a configured allowed root.
• Symlink escapes, traversal, special files, unsupported extensions, oversized files, excessive directory depth, file count, and aggregate size are rejected.
• All existing rules load successfully.
• Full `pytest` passes.

One correction worth enforcing firmly: an empty allowed-root list cannot remain “allow everything” if this is meant to be security hardening. That is compatibility theater, not containment.
