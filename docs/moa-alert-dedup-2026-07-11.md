# LOTL-Detector Alert Deduplication, Suppression, and Lifecycle Design

This design is grounded in the current repository:

- Persistence entry point: `core/database.py:110`
- Alert model: `core/engine.py:46`
- Event model: `collectors/base.py:13`
- API routes: `api/server.py:167`
- Configuration: `core/config.py:18`

The central design decision is:

Each `alerts` row represents one grouped alert episode. Repeated matching events inside the deduplication window update that row. A new episode is created after the window expires, or when the previous episode is closed.

This preserves meaningful recurrence without producing one row per event.

## 1. Fingerprint algorithm

### 1.1 Fingerprint fields

Use these fields in this exact order:

1. Fingerprint schema version
2. `alert.rule_id`
3. `event.platform`
4. Normalized host identity
5. Normalized process name
6. Normalized command line
7. Normalized user
8. Normalized parent process name

Canonical payload:

```json
{
  "v": 1,
  "rule_id": "LOTL-WIN-001",
  "platform": "windows",
  "host": "workstation-01",
  "process_name": "powershell.exe",
  "command_line": "powershell.exe -nop -c whoami",
  "user": "domain\\alice",
  "parent_process_name": "winword.exe"
}
```

Serialize with:

```python
json.dumps(
    payload,
    sort_keys=True,
    separators=(",", ":"),
    ensure_ascii=False,
)
```

Hash the UTF-8 encoded serialization with full SHA-256:

```python
hashlib.sha256(canonical_json.encode("utf-8")).hexdigest()
```

Store all 64 lowercase hexadecimal characters. Do not truncate the hash. Storage savings are negligible, while the full digest avoids unnecessary collision handling.

### 1.2 Why these fields are included

`rule_id`

Different rules represent different detection semantics, even when they match the same event.

`platform`

Prevents Windows and Linux command interpretations from being grouped.

`host`

Prevents identical commands on different endpoints from becoming one alert. This is also required for per-host suppression.

`process_name`

Distinguishes identical command text executed by different interpreters or binaries.

`command_line`

Represents the suspicious activity being grouped.

`user`

Different users normally require separate investigation.

`parent_process_name`

Parent context can materially change the meaning of an execution. PowerShell launched by an interactive shell is not equivalent to PowerShell launched by Word.

### 1.3 Fields explicitly excluded

Exclude:

- `timestamp`
- `process_id`
- `parent_process_id`
- `working_directory`
- `raw_data`
- `score`
- `severity`
- `risk_band`
- `rule_name`
- `description`
- `response`
- `mitre_attack`
- `scoring_version`

PIDs, timestamps, and raw event data are ephemeral. Rule metadata and scoring outputs can change between software versions without changing the underlying detection identity.

### 1.4 Normalization rules

All fields must be Unicode strings normalized using NFC.

Common text normalization:

```python
def normalize_text(value):
    value = "" if value is None else str(value)
    value = unicodedata.normalize("NFC", value)
    return value.strip()
```

Platform:

```python
platform = normalize_text(event.platform).casefold()
```

Process names:

Reuse `normalize_process_name()` from `core/engine.py:20`, then:

- Windows: lowercase through existing behavior
- Linux: preserve case
- macOS: use `casefold()` because standard macOS volumes are commonly case-insensitive

Parent process names use the same algorithm.

User:

- Trim leading and trailing whitespace
- Collapse internal whitespace
- Windows: `casefold()`
- Linux and macOS: preserve case
- Do not remove domain prefixes

Host:

Extract into a first-class `host` value during event ingestion. Use the following precedence:

1. `raw_data["hostname"]`
2. `raw_data["host"]`
3. `raw_data["computer_name"]`
4. `raw_data["Computer"]`
5. Empty string if unavailable

Normalize with `strip().rstrip(".").casefold()`.

Per-host suppression must return HTTP 422 if host identity is empty. Silently treating an unknown host as global would be dangerous.

Command line:

1. Convert missing values to an empty string
2. Normalize Unicode to NFC
3. Trim leading and trailing whitespace
4. Collapse runs of whitespace outside quoted strings to one ASCII space
5. Preserve quoted content and argument order
6. Windows: `casefold()`
7. Linux and macOS: preserve case

Do not:

- Remove numbers
- Remove paths
- Remove IP addresses
- Remove timestamps embedded in arguments
- Sort arguments
- Resolve environment variables
- Strip quotes globally
- Parse shell semantics

Those transformations can merge materially different commands.

Example:

```text
PowerShell.EXE    -NoP   -C "whoami /all"
```

Windows normalized value:

```text
powershell.exe -nop -c "whoami /all"
```

### 1.5 Fingerprint versioning

Add `fingerprint_version INTEGER NOT NULL DEFAULT 1`.

Any future normalization change must increment the version. Never silently change version 1 behavior, because that makes fingerprints unstable across upgrades.

## 2. Deduplication model and upsert logic

### 2.1 Episode semantics

A duplicate is an incoming alert that:

- Has the same fingerprint
- Falls within the configured sliding deduplication window
- Matches an alert in an open lifecycle state

Open states:

- `new`
- `acknowledged`
- `investigating`

Closed states are not deduplication targets:

- `resolved`
- `false_positive`

A repeat of a resolved or false-positive alert creates a new `new` episode. This prevents recurrence from being hidden inside closed work.

### 2.2 Time semantics

Use `alert.timestamp`, which currently comes from `event.timestamp`, for:

- `first_seen`
- `last_seen`
- Deduplication window evaluation

Use database UTC time for:

- `created_at`
- `updated_at`
- Suppression creation
- Suppression expiry
- Audit timestamps

Require timezone-aware event timestamps internally. If collectors currently produce naive timestamps, interpret them according to collector configuration and convert to UTC before persistence.

### 2.3 Out-of-order event handling

Find an existing episode when the incoming event timestamp is no farther than the dedup window from the episode interval:

```text
existing.last_seen >= incoming_timestamp - window
AND
existing.first_seen <= incoming_timestamp + window
```

This handles modestly out-of-order log ingestion.

After a duplicate:

```text
first_seen = minimum(existing.first_seen, incoming_timestamp)
last_seen = maximum(existing.last_seen, incoming_timestamp)
```

This is a sliding window. Continuous activity can keep one episode open. That is intentional, since the occurrence count and time span communicate the activity burst.

### 2.4 Transactional SQL flow

SQLite cannot express this episode selection safely as a conventional unique-key upsert because multiple historical episodes may share the same fingerprint.

Use one `BEGIN IMMEDIATE` transaction:

```sql
BEGIN IMMEDIATE;

SELECT id, occurrence_count, first_seen, last_seen, state
FROM alerts
WHERE fingerprint = ?
  AND state IN ('new', 'acknowledged', 'investigating')
  AND last_seen >= ?
  AND first_seen <= ?
ORDER BY last_seen DESC
LIMIT 1;
```

Parameters:

```text
fingerprint
incoming_timestamp minus dedup_window
incoming_timestamp plus dedup_window
```

If a candidate exists:

```sql
UPDATE alerts
SET occurrence_count = occurrence_count + 1,
    first_seen = MIN(first_seen, ?),
    last_seen = MAX(last_seen, ?),
    timestamp = MAX(timestamp, ?),
    score = MAX(score, ?),
    updated_at = CURRENT_TIMESTAMP,
    last_event_data = ?
WHERE id = ?;
```

The existing `event_data` remains the representative first occurrence. `last_event_data` holds the most recent occurrence.

Then optionally store a bounded duplicate sample:

```sql
INSERT INTO alert_occurrences (
    alert_id,
    event_timestamp,
    process_id,
    parent_process_id,
    working_directory,
    event_data
)
SELECT ?, ?, ?, ?, ?, ?
WHERE (
    SELECT COUNT(*)
    FROM alert_occurrences
    WHERE alert_id = ?
) < ?;
```

If no candidate exists, insert a new grouped alert:

```sql
INSERT INTO alerts (
    rule_id,
    rule_name,
    severity,
    score,
    platform,
    host,
    process_name,
    command_line,
    user,
    parent_process_name,
    timestamp,
    mitre_attack,
    description,
    response,
    event_data,
    last_event_data,
    fingerprint,
    fingerprint_version,
    occurrence_count,
    first_seen,
    last_seen,
    state,
    created_at,
    updated_at
) VALUES (
    ?, ?, ?, ?, ?, ?, ?, ?, ?, ?,
    ?, ?, ?, ?, ?, ?,
    ?, 1, 1, ?, ?, 'new',
    CURRENT_TIMESTAMP, CURRENT_TIMESTAMP
);
```

Commit only after the alert update and occurrence sample insert succeed.

### 2.5 Score and metadata behavior

On duplicate:

- `score`: retain the maximum observed score
- `severity`: retain the severity on the grouped alert
- `rule_name`: retain the current row value
- `event_data`: retain the first occurrence
- `last_event_data`: replace with the newest received occurrence
- `occurrence_count`: always increment, even when sample storage reaches its limit

The occurrence count is never capped. `max_occurrence_samples` caps only rows in `alert_occurrences`. Capping the actual count would make the dashboard misleading.

## 3. Suppression schema and enforcement

### 3.1 Suppression semantics

A suppression applies to one fingerprint.

Scopes:

- `global`: the fingerprint is suppressed on every host
- `host`: the fingerprint is suppressed only when `host` equals `scope_value`

A fingerprint already includes host to prevent cross-host deduplication. For global suppression, derive and store a second fingerprint named `activity_fingerprint` that excludes host.

Use:

- `fingerprint` for alert grouping and host-scoped suppression
- `activity_fingerprint` for global suppression

This avoids trying to reconstruct a hostless fingerprint from an existing digest.

### 3.2 Suppressions table

```sql
CREATE TABLE suppressions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    fingerprint TEXT NOT NULL,
    fingerprint_version INTEGER NOT NULL DEFAULT 1,
    scope TEXT NOT NULL
        CHECK (scope IN ('global', 'host')),
    scope_value TEXT,
    author TEXT NOT NULL,
    reason TEXT NOT NULL,
    starts_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    expires_at DATETIME NOT NULL,
    active INTEGER NOT NULL DEFAULT 1
        CHECK (active IN (0, 1)),
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    revoked_at DATETIME,
    revoked_by TEXT,
    revoke_reason TEXT,
    CHECK (
        (scope = 'global' AND scope_value IS NULL)
        OR
        (scope = 'host' AND scope_value IS NOT NULL)
    ),
    CHECK (expires_at > starts_at)
);
```

For a global suppression:

- `fingerprint` contains the hostless `activity_fingerprint`
- `scope_value` is `NULL`

For a host suppression:

- `fingerprint` contains the alert fingerprint
- `scope_value` contains the normalized host

### 3.3 Suppression enforcement algorithm

Check suppression after fingerprints are computed and before deduplication or insertion:

```sql
SELECT id
FROM suppressions
WHERE active = 1
  AND starts_at <= CURRENT_TIMESTAMP
  AND expires_at > CURRENT_TIMESTAMP
  AND (
      (
          scope = 'global'
          AND fingerprint = ?
      )
      OR
      (
          scope = 'host'
          AND fingerprint = ?
          AND scope_value = ?
      )
  )
ORDER BY
    CASE scope WHEN 'host' THEN 0 ELSE 1 END,
    expires_at DESC
LIMIT 1;
```

Parameters:

1. `activity_fingerprint`
2. `fingerprint`
3. normalized host

If matched:

- Do not insert or update an alert
- Increment `suppressions.match_count`
- Set `suppressions.last_matched_at`
- Insert an audit event with action `suppression_match`
- Return a persistence result with `outcome="suppressed"`

Add these operational columns:

```sql
ALTER TABLE suppressions
ADD COLUMN match_count INTEGER NOT NULL DEFAULT 0;

ALTER TABLE suppressions
ADD COLUMN last_matched_at DATETIME;
```

Suppression hits can be frequent. Audit rows should be aggregated per suppression per configurable audit interval, such as five minutes, rather than creating 100 identical audit rows. The suppression's `match_count` remains exact.

Do not delete expired suppressions. Expiry is part of the historical record.

Revocation sets:

```sql
UPDATE suppressions
SET active = 0,
    revoked_at = CURRENT_TIMESTAMP,
    revoked_by = ?,
    revoke_reason = ?
WHERE id = ?
  AND active = 1;
```

## 4. Alert lifecycle state machine

### 4.1 States

`new`

Generated and not yet reviewed.

`acknowledged`

An analyst has seen and accepted ownership of the alert.

`investigating`

Active investigation is in progress.

`resolved`

Investigation completed and activity handled or determined benign without declaring the detection incorrect.

`false_positive`

The rule matched, but the detection was judged incorrect or non-actionable in this context.

### 4.2 Allowed transitions

| Current | Allowed next states |
|---|---|
| `new` | `acknowledged`, `investigating`, `resolved`, `false_positive` |
| `acknowledged` | `new`, `investigating`, `resolved`, `false_positive` |
| `investigating` | `new`, `acknowledged`, `resolved`, `false_positive` |
| `resolved` | `new`, `investigating` |
| `false_positive` | `new`, `investigating` |

A transition to the same state is rejected with HTTP 409. This prevents meaningless audit entries.

Reopening from `resolved` or `false_positive` requires a non-empty reason.

Transitioning to `resolved` or `false_positive` also requires a non-empty reason.

### 4.3 Authorization

All state-changing endpoints require the existing bearer API key.

This is a single-user deployment, so no RBAC is required. The request still requires an `author` string for accountability. The server must not trust `author` as authentication. Authentication remains the API key.

Recommended default author:

```text
local-analyst
```

### 4.4 State metadata columns

Add:

- `state`
- `state_changed_at`
- `state_changed_by`
- `resolution_reason`

`resolution_reason` is populated for `resolved` and `false_positive`, and cleared when reopened.

## 5. Audit trail

### 5.1 Schema

```sql
CREATE TABLE alert_audit_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    action TEXT NOT NULL CHECK (
        action IN (
            'state_change',
            'suppression_created',
            'suppression_revoked',
            'suppression_match'
        )
    ),
    target_type TEXT NOT NULL CHECK (
        target_type IN ('alert', 'suppression', 'fingerprint')
    ),
    target_id INTEGER,
    alert_id INTEGER,
    suppression_id INTEGER,
    fingerprint TEXT,
    previous_state TEXT,
    new_state TEXT,
    author TEXT NOT NULL,
    reason TEXT NOT NULL,
    details TEXT,
    FOREIGN KEY (alert_id) REFERENCES alerts(id),
    FOREIGN KEY (suppression_id) REFERENCES suppressions(id)
);
```

`details` is JSON encoded as text. Example:

```json
{
  "scope": "host",
  "scope_value": "workstation-01",
  "expires_at": "2026-07-12T10:00:00Z"
}
```

The audit table is application-level append-only:

- No update method
- No delete endpoint
- No cascade deletion from alerts or suppressions

Deduplication itself does not require one audit row per hit. Exact occurrence history belongs in `alert_occurrences`; lifecycle and analyst actions belong in the audit trail.

### 5.2 Atomicity

State change and audit insertion must share one transaction:

```sql
BEGIN IMMEDIATE;

SELECT state
FROM alerts
WHERE id = ?;

UPDATE alerts
SET state = ?,
    state_changed_at = CURRENT_TIMESTAMP,
    state_changed_by = ?,
    resolution_reason = ?,
    updated_at = CURRENT_TIMESTAMP
WHERE id = ?;

INSERT INTO alert_audit_log (
    action,
    target_type,
    target_id,
    alert_id,
    fingerprint,
    previous_state,
    new_state,
    author,
    reason,
    details
) VALUES (
    'state_change',
    'alert',
    ?,
    ?,
    ?,
    ?,
    ?,
    ?,
    ?,
    ?
);

COMMIT;
```

Suppression creation and revocation follow the same pattern.

### 6.3 Fingerprint backfill

Fingerprint calculation cannot be done safely in standard SQLite because it requires JSON parsing, normalization, and SHA-256.

Migration Python must:

1. Select rows where `fingerprint IS NULL`
2. Parse `event_data`
3. Reconstruct the Event-compatible fields
4. Extract host from `event_data.raw_data`
5. Calculate both fingerprints
6. Update the row by ID
7. Commit in batches of 500

Do not merge existing rows during migration. Existing alerts remain separate episodes with `occurrence_count = 1`. Automatically merging historical rows risks changing analyst-visible evidence and IDs.

## 8. Configuration changes

Add to `DEFAULT_CONFIG` in `core/config.py`:

```yaml
alerts:
  dedup_enabled: true
  dedup_window_seconds: 3600
  fingerprint_version: 1
  max_occurrence_samples: 100
  suppression_default_seconds: 86400
  suppression_max_seconds: 2592000
  suppression_audit_interval_seconds: 300
```

Semantics:

`dedup_enabled`

When false, every non-suppressed match creates a new episode with occurrence count 1.

`dedup_window_seconds`

Sliding episode window. Recommended default: 3600 seconds.

`fingerprint_version`

Must match the implemented fingerprint version. Startup must fail if configured to an unsupported version.

`max_occurrence_samples`

Maximum rows stored in `alert_occurrences` per alert. The total `occurrence_count` remains uncapped.

`suppression_default_seconds`

Applied when the API request omits duration.

`suppression_max_seconds`

Reject longer requests with HTTP 400. A finite limit avoids forgotten permanent suppressions.

`suppression_audit_interval_seconds`

Minimum interval between aggregated `suppression_match` audit entries for the same suppression.

Validation:

- Dedup window: integer from 1 to 604800
- Fingerprint version: exactly 1 initially
- Maximum samples: integer from 0 to 10,000
- Default suppression: integer from 60 to maximum suppression
- Maximum suppression: integer from 60 to 31,536,000
- Audit interval: integer from 0 to 86,400

A permanent suppression is deliberately excluded. Analysts can renew a suppression, which creates visible audit history.

## 9. Concrete Python pseudocode

### 9.1 Fingerprint generation

```python
import hashlib
import json
import re
import unicodedata

from core.engine import normalize_process_name


def normalize_string(value):
    value = "" if value is None else str(value)
    return unicodedata.normalize("NFC", value).strip()


def normalize_command_line(command_line, platform):
    value = normalize_string(command_line)

    output = []
    quote = None
    pending_space = False

    for character in value:
        if character in ("'", '"'):
            if quote is None:
                quote = character
            elif quote == character:
                quote = None

            if pending_space and output:
                output.append(" ")
            pending_space = False
            output.append(character)
            continue

        if character.isspace() and quote is None:
            pending_space = True
            continue

        if pending_space and output:
            output.append(" ")
        pending_space = False
        output.append(character)

    normalized = "".join(output).strip()

    if platform in ("windows", "macos"):
        normalized = normalized.casefold()

    return normalized


def extract_host(event):
    raw = event.raw_data or {}

    for key in ("hostname", "host", "computer_name", "Computer"):
        value = raw.get(key)
        if value:
            return normalize_string(value).rstrip(".").casefold()

    return ""


def canonical_fingerprint_payload(alert, include_host):
    event = alert.event
    platform = normalize_string(event.platform).casefold()

    process_name = normalize_process_name(
        event.process_name,
        platform,
    )
    parent_name = normalize_process_name(
        event.parent_process_name or "",
        platform,
    )

    if platform == "macos":
        process_name = process_name.casefold()
        parent_name = parent_name.casefold()

    user = normalize_string(event.user)
    if platform == "windows":
        user = user.casefold()

    payload = {
        "v": 1,
        "rule_id": normalize_string(alert.rule_id),
        "platform": platform,
        "process_name": process_name,
        "command_line": normalize_command_line(
            event.command_line,
            platform,
        ),
        "user": user,
        "parent_process_name": parent_name,
    }

    if include_host:
        payload["host"] = extract_host(event)

    return payload


def hash_payload(payload):
    serialized = json.dumps(
        payload,
        sort_keys=True,
        separators=(",", ":"),
        ensure_ascii=False,
    )
    return hashlib.sha256(
        serialized.encode("utf-8")
    ).hexdigest()


def calculate_fingerprints(alert):
    return {
        "fingerprint": hash_payload(
            canonical_fingerprint_payload(alert, include_host=True)
        ),
        "activity_fingerprint": hash_payload(
            canonical_fingerprint_payload(alert, include_host=False)
        ),
        "host": extract_host(alert.event),
        "fingerprint_version": 1,
    }
```

def find_active_suppression(
    cursor,

    activity_fingerprint,
    host,

        FROM suppressions
        WHERE active = 1

              (
                  scope = 'global'
                  AND fingerprint = ?
              )
              OR
              (
                  scope = 'host'
                  AND fingerprint = ?

### 9.4 Deduplicating save

```python
def save_alert(self, alert):
    fingerprints = calculate_fingerprints(alert)
    event_time = ensure_utc(alert.timestamp)
    window = timedelta(
        seconds=self.config["alerts"]["dedup_window_seconds"]
    )
    event_json = json.dumps(alert.event.to_dict())

    cursor = self.connection.cursor()

    try:
        cursor.execute("BEGIN IMMEDIATE")

        suppression = find_active_suppression(
            cursor=cursor,
            fingerprint=fingerprints["fingerprint"],
            activity_fingerprint=fingerprints["activity_fingerprint"],
            host=fingerprints["host"],
        )

        if suppression is not None:
            record_suppression_match(
                cursor=cursor,
                suppression=suppression,
                fingerprint=fingerprints["fingerprint"],
                event_time=event_time,
                audit_interval_seconds=self.config["alerts"][
                    "suppression_audit_interval_seconds"
                ],
            )
            self.connection.commit()
            return AlertSaveResult(
                outcome="suppressed",
                alert_id=None,
                suppression_id=suppression["id"],
            )

        existing = None

        if self.config["alerts"]["dedup_enabled"]:
            cursor.execute(
                """
                SELECT id, occurrence_count
                FROM alerts
                WHERE fingerprint = ?
                  AND state IN (
                      'new',
                      'acknowledged',
                      'investigating'
                  )
                  AND last_seen >= ?
                  AND first_seen <= ?
                ORDER BY last_seen DESC
                LIMIT 1
                """,
                (
                    fingerprints["fingerprint"],
                    event_time - window,
                    event_time + window,
                ),
            )
            existing = cursor.fetchone()

        if existing is not None:
            cursor.execute(
                """
                UPDATE alerts
                SET occurrence_count = occurrence_count + 1,
                    first_seen = MIN(first_seen, ?),
                    last_seen = MAX(last_seen, ?),
                    timestamp = MAX(timestamp, ?),
                    score = MAX(score, ?),
                    last_event_data = ?,
                    updated_at = CURRENT_TIMESTAMP
                WHERE id = ?
                """,
                (
                    event_time,
                    event_time,
                    event_time,
                    alert.score,
                    event_json,
                    existing["id"],
                ),
            )

            store_occurrence_sample_if_below_limit(
                cursor=cursor,
                alert_id=existing["id"],
                event=alert.event,
                limit=self.config["alerts"][
                    "max_occurrence_samples"
                ],
            )

            cursor.execute(
                """
                SELECT occurrence_count
                FROM alerts
                WHERE id = ?
                """,
                (existing["id"],),
            )
            occurrence_count = cursor.fetchone()[0]

            self.connection.commit()
            return AlertSaveResult(
                outcome="deduplicated",
                alert_id=existing["id"],
                occurrence_count=occurrence_count,
            )

        alert_id = insert_grouped_alert(
            cursor=cursor,
            alert=alert,
            fingerprints=fingerprints,
            event_json=event_json,
            event_time=event_time,
        )

        self.connection.commit()
        return AlertSaveResult(
            outcome="created",
            alert_id=alert_id,
            occurrence_count=1,
        )

    except Exception:
        self.connection.rollback()
        raise
```

### 9.5 State transition logic

```python
ALLOWED_TRANSITIONS = {
    "new": {
        "acknowledged",
        "investigating",
        "resolved",
        "false_positive",
    },
    "acknowledged": {
        "new",
        "investigating",
        "resolved",
        "false_positive",
    },
    "investigating": {
        "new",
        "acknowledged",
        "resolved",
        "false_positive",
    },
    "resolved": {
        "new",
        "investigating",
    },
    "false_positive": {
        "new",
        "investigating",
    },
}


def change_alert_state(
    connection,
    alert_id,
    new_state,
    author,
    reason,
):
    if new_state not in ALLOWED_TRANSITIONS:
        raise InvalidState(new_state)

    if not author.strip():
        raise ValidationError("author is required")

    cursor = connection.cursor()

    try:
        cursor.execute("BEGIN IMMEDIATE")

        cursor.execute(
            """
            SELECT id, state, fingerprint
            FROM alerts
            WHERE id = ?
            """,
            (alert_id,),
        )
        alert = cursor.fetchone()

        if alert is None:
            raise AlertNotFound(alert_id)

        previous_state = alert["state"]

        if new_state == previous_state:
            raise InvalidTransition(
                previous_state,
                new_state,
            )

        if new_state not in ALLOWED_TRANSITIONS[previous_state]:
            raise InvalidTransition(
                previous_state,
                new_state,
            )

        reason = reason.strip()

        if (
            new_state in {"resolved", "false_positive"}
            or previous_state in {"resolved", "false_positive"}
        ) and not reason:
            raise ValidationError(
                "reason is required when closing or reopening an alert"
            )

        resolution_reason = (
            reason
            if new_state in {"resolved", "false_positive"}
            else None
        )

        cursor.execute(
            """
            UPDATE alerts
            SET state = ?,
                state_changed_at = CURRENT_TIMESTAMP,
                state_changed_by = ?,
                resolution_reason = ?,
                updated_at = CURRENT_TIMESTAMP
            WHERE id = ?
            """,
            (
                new_state,
                author,
                resolution_reason,
                alert_id,
            ),
        )

        cursor.execute(
            """
            INSERT INTO alert_audit_log (
                action,
                target_type,
                target_id,
                alert_id,
                fingerprint,
                previous_state,
                new_state,
                author,
                reason,
                details
            ) VALUES (
                'state_change',
                'alert',
                ?,
                ?,
                ?,
                ?,
                ?,
                ?,
                ?,
                '{}'
            )
            """,
            (
                alert_id,
                alert_id,
                alert["fingerprint"],
                previous_state,
                new_state,
                author,
                reason,
            ),
        )

        connection.commit()

    except Exception:
        connection.rollback()
        raise
```
